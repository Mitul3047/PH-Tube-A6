Discuss the scope of var, let, and const

  • var: The scope of a var variable is function scope or global scope. This means that a var variable can be accessed from anywhere within the function where it is declared, or from anywhere in the global scope.
  • let: The scope of a let variable is block scope. This means that a let variable can only be accessed from within the block where it is declared.
  • const: The scope of a const variable is block scope. This means that a const variable can only be accessed from within the block where it is declared. However, once a const variable is assigned a value, that value cannot be changed.

Tell us the use cases of null and undefined

  • null: The value null is used to indicate that a variable does not have a value. For example, you could use null to indicate that a user has not entered a value into a form field.
  • undefined: The value undefined is used to indicate that a variable has not been assigned a value yet. For example, the value of a variable is undefined immediately after it is declared.
